Here is an example of the output of PPL LIST TAXIS.
The data in question has a time range (from "SHO GRID") of:

 TIME      TIME              1781 r   03-JAN-1979 00:00    23-MAY-2003 00:00

Note that PPL LIST TAXIS only functions AFTER a plot has been made!
For the above data, plotted without specifying anything about the 
time range, this command shows:

yes? PPL LIST TAXIS


         TAXIS QUALITIES
         LO= 8.77E+03 DT= 6.00E+01 CSIZE= 1.00E-01 TYPE= TAXIS3 VANG= 0.00E+00
          TOP AXIS= ON  BOTTOM AXIS= ON 
           TMIN=19790101000000  0:00 JAN  1,1979    
           TMAX=20030601000000  0:00 JUN  1,2003    
         TSTART=19771231115734 11:57 DEC 31,1977    
         NMTC= -1 LABEL= -3 IFDB=  0 ISDB=  2 TAXIS= XAXIS IAUTOT= ON 

TMIN and TMAX are the actual plot ranges. How TSTART was chosen I 
have no idea, but as is typical, it is about 1 year before the 
actual left edge of the plot. In the first line of output, the
useful values are:

LO= 8.77E+03, which is the user-unit value (hours) of the left edge.
Note that LO is a rounding of `365.24*24` = 8765.8.

DT= 6.00E+01, which tells you that one time axis unit is 60 minutes.
(I was wrong in the previous example. Ferret actually works in minutes,
I think, but somehow the unit actually used in many plots extending
over months or years is hours). DT tells you that.

The rest of the information is incomprehensible to me.

Billy K
