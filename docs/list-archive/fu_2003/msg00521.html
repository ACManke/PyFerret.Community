Hi Kevin and linux users,

I've change the ftp path: it's now
ftp://ftp.ifremer.fr/pub/ifremer/cersat/linux/debian/dists/stable/contrib/binary-i386/

so debian user should add
deb ftp://ftp.ifremer.fr/pub/ifremer/cersat/linux/debian/ stable contrib
to their /etc/apt/sources.list

I've also use alien to generate rpm ( v 4.0.3 ) from debian packages. they are avialable on ftp://ftp.ifremer.fr/pub/ifremer/cersat/linux/rpm
I've tested (well, it was not a full testing) rpm on redhat 7.2 8.0 9.0 and mandrake 8.0 and 9.0.
so users of this flavor of linux should install ferret with 'rpm -Uhv ferret-551-1-2.i386.rpm', as usual on those systems.


I've automated package building with a cron, so if the next ferret version (5.53) is avialable on
ftp://www.ferret.noaa.gov/pub/linux/fer_executables.tar.Z, the package building proccess should /hopefully/ generate deb and rpm version for ferret 5.53, while I'm on holiday :)


Kevin, will ferret v 5.53 will be a gnu f77 build ? will it be linked whith libncurses instead of libtermcap ?
