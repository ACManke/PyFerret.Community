If the levels are being specified then there is an easier
way to do it: 

fill/lev=(lev1,lev2,increment,-3) variable
ppl contour/over

The fourth argument to the lev qualifier lets you specify
the number of digits in the contour labels:

0-9: number of digits after the decimal point
-1: contour label plotted as an integer
-3: no contour label drawn

Seems to me that one often sees published contour plots with 
too many digits (trailing zeroes).
