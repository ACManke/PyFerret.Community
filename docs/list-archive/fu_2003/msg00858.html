I think I may have found a possible bug in Ferret. It looks like it's something to do with caching results.

If I use ferret in -gif mode and use a journal script that has a a repeat loop in it (which may not be significant), at some point in the loop (let's say l=30) I generate a plot that has a small difference in the fields from two different netcdf files I am comparing. Further generated plots from the repeat loop show the same difference i.e. l=31:40

However, if I then go into Ferret and do the plot normally (no -gif) and by hand, no script, although plot l=30 shows the difference, further values of l don't (l=31:40).

Is this a known problem? I can send example scripts/files but I thought I'd check with the list first that this is known about.

I'm using Ferret v5.51 on Solaris 8. I am going to try the latest version of Ferret but I don't see any mention of such a bug in the release notes for v5.53.

I'm also having trouble downloading the latest Ferret. ftp keeps returning, Illegal Port Command. Anyone have any ideas what that means?
