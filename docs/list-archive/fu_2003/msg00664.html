Hi all,

After looking the mail archive concerning this subject,
I believe that I'm facing a different type of ferret 
behaviour related with the use of data irregularly spaced
in time. 

The idea is to plot a series of lon/lat vectors, unevenly
spaced on time, over 2D fields. In my case, SST fronts extracted
from synoptic satellite images, over monthly SST averages. 

The attached scripts reproduce the ferret behaviour using
abstract data because the data files I'm using are too big
to attach:

yes? go vecs_gen  ! generate abstract fronts

and,

yes? go vecs_overlay ! now, plot the fronts over the monthly field

the script runs OK for the first time steps and then continues
with the error message:
 **TMAP ERR: limit on storage for coordinates has been reached
             MAX= 250000
but the vectors are not plotted.

Its interesting to note that if the script is interrupted and 
restarted afterwards, the error message comes right after the 
first time step.

It seems that some coordinate info is acumulating into 
memory at each time step and may be there is a command
to clear it. Any hints will be appreciated.

Regards, 

Paulo.
