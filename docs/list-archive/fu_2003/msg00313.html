Hi Nicolas -

Unfortunately, this turns out to be a limit due to the Nag f95 compiler, which is what we have been using to build Ferret on Linux systems. This is why you've just noticed it switching from SGI to Linux Ferret, though the limit has actually always existed in the Linux version of Ferret.

However, we have recently started to build Ferret using the GNU g77 Fortran compiler. Ferret built using this compiler has no such limitation. If you are interested in downloading a beta version of Ferret built w/ the g77 compiler, you can get to it from:

http://ferret.wrc.noaa.gov/Ferret/Downloads/beta_releases.html


Let me know if you have any questions -

Kevin

Dear Ferreters

I recently updated the version of ferret I am working with,
from version v5.40 (beta) (with IRIX OS) to v5.51 (with linux).

With the older version I could read ascci files with lines containing
about 1400 ascii characters such as :

0.3683E+02 0.3680E+02 0.3675E+02 0.3669E+02 0.3664E+02 0.3659E+02 0.3657E+02 0.3655E+02 0.3652E+02 0.3650E+02 0.3649E+02 0.3648E+02 0.3647E+02 ....

Is there a way to solve this problem ?
