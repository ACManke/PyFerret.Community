Jack,

We will have a fix for this bug in the v4.9 package soon.  In the mean-time you
can work around it by removing metafile item 51 (polyline representation) from
the metafiles you create, and processing the result in the usual way with
Fprint:

1) grep -v "^ 51" metafile.plt > fixed_metafile.plt
2) Fprint fixed_metafile.plt

If this temporary fix doesn't solve the problem for you please let me know.

	Regards,

	Jerry Davison
