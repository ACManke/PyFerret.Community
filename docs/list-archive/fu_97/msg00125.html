I am the one who installed ferret at Ye's site and all the ferret
programs are called here with a single shell script that ends like
this:

exec $FER_DIR/bin/`basename $0` $*

I just tried to change this line to

exec $FER_DIR/bin/`basename $0` "$*"

and this solved the problem with Fpalette, but the solution is worse
than the problem since ferret can no longer be run.

Ye, I guess the best solution for the time being is to copy this script
(/usr/local/bin/Fpalette) to your account, add the double quotes like
above and access it through an alias, for example:

alias Fpalette ~/bin/Fpalette
