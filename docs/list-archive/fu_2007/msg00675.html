On Dec 10, 2007 3:10 PM, Fabian Lienert < cccma-student-003@xxxxxxxx> wrote:
Hi Domingos,

I don't see how you used eof_stat, but the error message tells
you that there is not enough RAM for the computation (RAM, not swap).

If the resolution of the grid of the input data is high, inverting such a matrix in order to
find the EOFs uses a lot of RAM.
Calculating the EOFs in the northern Pacific (20N,65N) for 330 time steps needs about 800 MB of RAM
if the data is on a T63 grid (lon = 256, lat = 192).
Higher resolved input data leads to the use of more RAM. This behavior is not linear.

Fabian
