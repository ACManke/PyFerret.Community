Hi Janeen,
There is not currently a way to define a variable in Ferret and keep the lower-case characters in the name on output; in Ferret v6.0 we can keep the case of variables that came in from a NetCDF file, and I think it should be possible for us to implement the same kind of logic for the names of user-defined variables; so look for that in a future release.
One solution for you would be to call NCO operations to fix up your file after Ferret has created it. These are available at http://nco.sourceforge.net/ 

Janeen wrote:

Hello All,
I'm using the list command in a Ferret script to write out a variable to a NetCDF file. The variable name is an input parameter to the script and consists of upper and lower case characters. The script re-grids the variable onto a different axis, then writes out the variable+suffix to the output NetCDF file. In writing the re-gridded variable out, the variable name is converted to all uppercase characters plus my suffix.
Does anyone know if there is a way to instruct Ferret not to change the case of the variable name? I'm using FERRET v5.80
