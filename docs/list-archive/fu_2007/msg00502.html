 Here is the error message/library I am missing:
ferret: error while loading shared libraries: libXp.so.6: cannot open shared object file: No such file or directory 

Steve
Well, this seems to be a bit tricky since the library is officially deprecated, but it appears there is a package you can get which will satisfy this reference. 

[roland@ip-208-109-127-60 ~]$ yum whatprovides libXp.so.6
Searching Packages:
Setting up repositories
Reading repository metadata in from local files
xorg-x11-deprecated-libs.i386 6.8.2-1.EL.18 base Matched from: 
libXp.so.6
You can try the "yum whatprovides" to see if it's installed on your system. If it is you might just need to add it to your LD_LIBRARY_PATH. 

If not, try installing it:
yum install xorg-x11-deprecated-libs.i386 Roland

Which shared library or libraries are missing? If it happens to be libstdc++.so.5 then you can install the compat-libstdc++33 package.
