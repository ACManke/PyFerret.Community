Hi Ferreters,

I'm not sure if this little problem has been reported
before. . . . When the last line of a go script doesn't
end with a newline, the line is not executed.  I think
it's better that this problem be fixed in the future.

In the following example, each of the two scripts
consists of a single line "say hello", but the first
one doesn't contain a newline and the second does,
which is the only difference:

>>>>>>>>>>>>>>>>>>
        NOAA/PMEL TMAP
        FERRET v6.07  
        Linux(g77) 2.4.21-32 - 08/29/07
        10-Oct-07 15:59     

yes? SP cat tmp.jnl
say helloyes? SP cat tmp2.jnl
say hello
yes? go tmp.jnl
yes? go tmp2.jnl
say hello
hello
yes? 
<<<<<<<<<<<<<<<<<<
