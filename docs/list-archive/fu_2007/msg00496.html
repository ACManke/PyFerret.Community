Hi Aurien,
           This is a typical problem when you use windows based text
files in Linux. There are two (many more exists!) easy solutions :

  1. On your Linux machine, check whether you have "dos2unix" command

         [user@machine]$ which dos2unix
         /usr/bin/dos2unix
         [user@machine]$
 
     If you see something like this, then try,

         [user@machine] dos2unix minoxNDEF2.jnl

     It will chage all windows related stuff (for example, carriage return)
        to that off native linux. You can use the resulting jnl file (exactly
        same file name) directly in Ferret.

  2. If you don't have access to dos2unix, then open minoxNDEF2.jnl in a 
        text viewer (say vim), copy all its content, open a new text file
        and paste to it. Now, you can use the new text (jnl) file in Ferret.

Let me know if you have any questions......
