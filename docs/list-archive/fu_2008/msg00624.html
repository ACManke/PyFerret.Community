All,

I thought I'd follow up on this for the benefit of the list archive.
(With thanks to those who responded to me.)

TIME is not a reserved word.  My input data set had TIME as a variable
and that was confusing things.  A clue that the regridding was not
working right is this output in the listing "regrid: ARRAYTYPE"; if
the regridding works it looks something like "regrid: 3600 sec on T@AVE".

I've changed the construction of my input datasets to not have a 
variable of TIME and the regridding is working now.

-Mike
