Ansley Manke wrote:

Hi Vincent,
There are probably a number of considerations here. Ferret doesn't have anything that specifically lets it make use of multiple processors, so unless you're running more than one Ferret job at once I don't know that that would have much impact. If you're not getting messages about Ferret not having enough memory to do commands, then Ferret is not running into limitations there. If you are able to get rid of any such errors with a SET MEMORY/SIZE= then this is not really a limitation either. The speed of the processor(s) will of course also have an effect. 

Anyone else have comments?
