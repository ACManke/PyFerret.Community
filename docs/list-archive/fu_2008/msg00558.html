Hi Ansley,

Thanks for your appreciation!

And an additional trick: The axis names are intentional. So we can do, for example,

use climatological_axes
can data climatological_axes
let sst_clim = sst[gt=tax_`sst,return=cal`@mod]

Hence one even needs not to know the calendar attribute a priori. This trick is pretty useful in some occasions.

David

David Wang wrote:
Hi Ferreters,

I created three additional pre-defined monthly climatological axes in the attached netcdf file. The purpose is to speed the common practice of computing monthly climatologies and anomalies when the original data are not defined on a Gregorian time axis. TAX_GREGORIAN is the same as MONTH_IRREG in climatological_axes.cdf; TAX_NOLEAP is for NOLEAP calendar axes, and TAX_360_DAY is for 360_DAY calendar axes.

Hope it helps,
David
