Hi Domingos,

>From the SHOW GRID output that you pasted in below it appears that the dataset in question lacks coordinate axes.  As-is, the "coordinates" are simply the i,j index values.  You can either work with it using the indices as the coordinates.   Or you can inform Ferret of the correct coordinates to use by means of the DEFINE AXIS command  (http://porter.pmel.noaa.gov:8980/Ferret/home/documentation/users-guide/commands-reference/DEFINE#_VPINDEXENTRY_1299)

Quite likely you can simply redefine the axes FAKEDIM5 and FAKEDIM4 using the DEFINE AXIS command.

    - Steve

Dear all,

I'm trying to access the Level 3 QuickSCAT gridded vectors data using OpenDap on ferret.
However, the grid is not organized in degrees of latitude and longitude. How can I handle this on Ferret?
Best,
Domingos.
