You might be able to convert it to netCDF using wgrib2: http://www.cpc.ncep.noaa.gov/products/wesley/wgrib2/

Or you might be able to read it as an OPeNDAP data source if you first serve it via the THREDDS Data Server: http://www.unidata.ucar.edu/software/thredds/current/tds/TDS.html

But there is no way as far as I know to read the file directly with Ferret.

The example you cite from the email archives is using the second method of reading the data via the TDS.
