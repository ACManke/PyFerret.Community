A kluge that sometimes conceals problems like this is to first SHADE, then PPL FILL/OVER. But this is strictly a kluge when you must have a plot. 

You might get some insight by LISTing a single column of data (profile) across that apparent gap. Is there a blank? 

Is the z-axis regular? If not, try also LISTing ZBOXLO and ZBOXHI for the profile. Does the top of one cell meet the bottom of the one above? If this is not consistent with what you think the profile depths are, then it could be how the depth axis is defined.
