Hi Hussen,
We update the land datasets using information from the Global Admministrative Areas database, at http://www.gadm.org.  It looks as though they do have updated boundaries for Eritrea, and we will work on adding those boundaries to the files used by "go land" and "go land_detail".  In the meantime, I have gone ahead and made a separate file, eritrea_0.nc, from the "level 0" data from that site, and I have attached that.

There are instructions in this FAQ,
http://ferret.pmel.noaa.gov/Ferret/faq/detailed-coastlines-and-borders/

for how to get data from GADM for use with Ferret.  Note that you want to use links for Level 0, level 1 or level 2 data at the bottom of the country page.

To use the file as an addition to "go land" or "go land_detail", here is an example,
yes? use etopo20
yes? shade/x=20:50/y=-10:30/lev=c/pal=land_sea rose
yes? go land thick " " black
yes? use eritrea_0.nc
yes? plot/vs/over/line/nolab/color=black lon,lat
The file from GADM includes many small islands offshore.  One could look at the data in the KMZ file and perhaps make a version which doesn't have that much detail, so that it would be more equivalent to the level of detail in "go land".
