Hi,

Saurabh and I have determined that the current release of PyFerret, version 7.5, corrects the behavior that was the problem here.

Just to recap some of the details about the character-size settings for colorbars:

set text/font= /isize= is used to set the size of all text, or of the text for different text-groupings on a page.   Its argument is not a character size, it is an integer to make the text bigger with values 1 through 4, or smaller with values -1 through -4.

PPL SHAKEY is used to customize colorbars, including the placement on the page, the width of the bar, and elements of the labeling including the size of the labels.  One of its arguments is the label-size setting, given in "inches".

So often, using Pyferret's SET TEXT will be enough to change the size of the the text in labels, but if using PPL SHAKEY for other customizations such as the placement of the colorbar itself, the size argument can be useful. 

-Ansley

On Thu, May 30, 2019 at 12:36 PM Ansley C. Manke <ansley.b.manke@xxxxxxxx> wrote:
Hi,

I'm not sure what problem you are having.  SHAKEY gives you a lot of control over the size of labels as well as other features of the colorbar.  The SET TEXT command with /ISIZE is intended as a simple short-cut way to increase or decrease the size of different label groups including the colorbar For example, try:

shakey for large labels:

use coads_climatology
shade/set/l=1 sst
ppl shakey 1,1,0.2
ppl shade

settings for larger, then smaller text using SET TEXT in pyferret:

set text/font=arial

set text/isize=+1 cbar
shade/l=1 sst

shade/l=1 sste=+3 cbar
shade/l=1 sst

set text/isize=-2 cbar
shade/l=1 sst
