Hi Saurabh,

On Sun, May 24, 2020 at 9:29 AM saurabh rathore <rohitsrb2020@xxxxxxxxx> wrote:

I think you can manage it by 5th and 6th arguments in the PPL Shakey Command.

ppl shakey 1, 1, 0.19, , 6, 6, 6.45, 6.65, 0.5, 3.6

0.19 is the size of the label in the color key

and 5th position (6) is the number of significant digits you want so eventually it will count that and put it with space.

Thanks!  I thought that that's a good idea, but I don't know why the following doesn't make any change:

yes? set data levitus_climatology
yes? fill/k=1/set temp
yes? ppl shakey 1, 1, 0.13, , , , 9.4, 9.6
yes? ppl fill
yes? fill/k=1/set temp
yes? ppl shakey 1, 1, 0.13, , 6, 6, 9.4, 9.6
yes? ppl fill

I don't know what klab_dig (the 5th argument) is supposed to do.  The following is what I found from somewhere a long time ago:

klab_dig is the number of significant digits (klab_dig > 0)
          or decimal places (klab_dig < 0) in the key.
          Default is 3.

Perhaps this issue has been discussed in this mailing list.  I haven't checked it yet.
