Hi,

The netCDF library is statically linked, and is kept up to date with each Ferret/PyFerret release.   So updating the Ferret installation should take care of a known, fixed issue in netCDF.

On 8/27/2020 8:56 PM, Ryo Furue wrote:
Ansley,

On Fri, Aug 28, 2020 at 6:21 AM Ansley C. Manke <ansley.b.manke@xxxxxxxx> wrote:
Just to close this thread, I can confirm that this is not an error written by Ferret or PyFerret code, but seems to have come from the netCDF library. That doesn't tell us whether it's an issue within netCDF, or whether the netCDF code has run into something wrong with the file.

I keep forgetting, but is the netCDF library statistically linked to the Ferret binary?  (I don't have the classical Ferret handy at the moment.)

As Russ suggested, this bug(*footnote*) in the netCDF library may have been fixed for some time.  If so, upgrading Ferret to the latest version may fix it.

If it's dynamically linked, you would want to upgrade the netCDF library installed on your computer.

Regards,
Ryo
