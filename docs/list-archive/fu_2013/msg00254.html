On 4/9/2013 12:37 AM, Hein Zelle wrote:

Dear Steve, Gary,


Ferret expects the CF auxiliary coordinates to be on the same 2D
horizontal coordinate system as the data variables.   Have a look at
using the RESHAPE function to wrap the 1D coordinate arrays onto a
2D coordinate system.

On 4/6/2013 7:27 AM, Gary Strand wrote:


Giving lon and lat the appropriate "axis" attributes doesn't
change anything about how FERRET sees the data.

Does Ferret do anything with the "axis" attribute and CF conventions
for curvilinear grids?  If so, that could be really useful for
visualizing WRF model data, but I wasn't aware of it yet.  Steve,
could you confirm if that's the case and if so, is there any
documentation on that?


Hi Hein,
It is on our to-do list that Ferret will pay attention to the 'coordinates' attribute, and automatically recognize curvilinear coordinate systems. That change has been awaiting the availability of the fast regridding code that the ESMF project has now produced. This capability is planned to be added within the next year. 

    - Steve



Kind regards,
      Hein
