Russ Fiedler wrote:

> Yes, nesting grave accents is tricky. You probably need to break things
> up into a few steps

This is something we run into a lot as well.  It can often be resolved
in multiple steps, but remains tricky.  Ideally I would like to have
bash syntax instead of grave accents:

$(variable[d=2])   instead of `variable[d=2]` .  The advantage is that
the first one can be nested.  This would probably be a major change to
the ferret parser, so I have no idea if that's feasible.
