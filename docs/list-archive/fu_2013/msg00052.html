Hi,
This is all about the command-line parsing.  As Ferret reads in each line, it's interpreting the characters -- what command is the user asking for? Ah, it's the LABEL command.  What are its arguments? Then it's looking for commas or spaces between the arguments. 

When quotes or other special characters are encountered, the parsing step is looking to see if there a variable to be evaluated, or is the text in the command indicating that a value of a symbol translated and then put into the command string? When special characters are encountered, like ^, that may indicate that something should be done like make a superscript.

Much of the handling of text for labels is in the PPLUS library which Ferret calls for graphics.  This means a certain amount of study of this part of the documentation.

A couple of useful references,
  • For putting ascii characters, such as ^, into a string for a label, http://ferret.pmel.noaa.gov/Ferret/faq/ascii-characters-in-labels
  • Search for the word "quote" in this PPLUS documentation, http://ferret.pmel.noaa.gov/Ferret/documentation/users-guide/appendix-b-pplus-guide/PPLUS-COMMAND-FILES  where it says, To have a single quote (') in the symbol or command line two single quotes must be used (''). To have a double quote (") in the command line two double quotes ("") are required.
So, Peter's command could be done this way, where between the C and the 1 are two single-quotes:
yes? label/nouser 6.5,-0.88,0,0,0.22 "C''1"

By the way, there is no /OVER qualifier for the LABEL command,
http://ferret.pmel.noaa.gov/Ferret/documentation/users-guide/appendix-b-pplus-guide/PPLUS-COMMAND-FILES#_VPID_651 but in this instance the /over is ignored.

-Ansley
