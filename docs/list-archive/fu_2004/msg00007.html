Hi Gary,

We ran into this a few months ago at GFDL.  It seems to be a system
buffering issue in Linux.  I don't know if there's been progress on fixing
it yet, but a work-around is to write a short wrapper script, e.g. in your
case a journal file "wrapper.jnl" that just contains "GO script.jnl".  
Then just do "ferret -gif < wrapper.jnl".  More details are below.
