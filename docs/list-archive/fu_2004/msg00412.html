Dear DuHui,
            The default unit used in ferret is cycles/day( in example with
navy winds) if your data is sampled at hours ,  you will get the axis values
in cycles per hour provided in  your netcdf file the definition of time
units is "hours". If you are using ascii file,define taxis units as hours or
days or months which ever you want. Once this is done unit problem is
solved.
example:
for a data sapled at every 24 hour from 2001 Jan to 2002 Jan

 define axis/t=1-jan2001:1-jan-2002:24/units=hours tax
 file/var=CO2  co2.dat
 let tco2=co2[gt=tax]
 let fftco2=ffta(tco2)
 plot fftco2


question NO.2  Box sixe is nothing but your total time length since ferret
treats every thing in terms of grid this terminology is used.

Niquist frequency: this is the highest frequency you can obtain from the
data you are using with a specified sampling interval  which is
1/2*N(del_T), where N is the total number of samples del_T is the sampling
interval(Total time=N*del_T) Boxsixe= N*det_t because ferret has already
taken in to account the time units.
Hope this helps.
