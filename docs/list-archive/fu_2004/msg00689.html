Hi David Muhati,
                  Steve Hankin explained the reason for almost a similar error
and i think it is relevant in your case also. Please have a look at it :
 http://ferret.pmel.noaa.gov/Ferret/Mail_Archives/fu_2001/msg00535.html

   What will be the total size of the resultant file ? I think if it is > 2GB
then the best method is to write a descriptor file so that you can access the
collection of individual files as a single data set. If you need any details
regarding descriptor files please refer in user manual :  Ch10 Sec4. CREATING
A MULTI-FILE NETCDF DATA SET. You can see lot of discussions about descriptor
files in Ferret Mail archive (go to http://ferret.pmel.noaa.gov/Ferret/
Mail_Archives/ferret_mail_archives.html and search for the key word "descriptor" )
