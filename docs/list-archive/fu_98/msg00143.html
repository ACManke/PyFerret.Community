> I am relatively new to Ferret and have a few questions about basemap
> usage:
> 
> 	a) Does "go basemap" always generate tick marks and lat/lon
> 	   labels (netcdf dataset)?
>   
> 	b) If (a) is true, does that mean this feature is suppressed
> 	   when "contour", "fill", or "shade" are invoked.
>   
> 	c) Can the user specify the tick mark/label interval? If so,
> 	   can the tick marks point inward instead of outward and can
> 	   you specify the sides (say left & bottom) to put them on?
> 
> 	d) it appears that for a polar stereographic plot you cannot
>            use "go polar_fland 20 gray" unless "go polar_2d fill. . ."
> 	   has been issued first; i.e., if "go polar_fland 20 gray" is
> 	   first a fatal error occurs. Consider the following:
> 
> (1)  set data EC80uvrhtz.nc
> (2)  cancel region/x
> (3)  set region/l=1
> (4)  set region/y=0:90
> (5)  set data EC80uvrhtz.nc
> (6)  set window/aspect=1:ax/size=.5
> (7)  go polar_2d fill/overlay/lev=(5200,5440,60)/overlay/title="500mb Heights"/pal=no_red HGT[Z=500,L=45]
> (8)  go polar_fland 20 gray 
> (9)  go polar_2d fill/overlay/lev=(5200,5440,60)/overlay/title="500mb Heights"/pal=no_red HGT[Z=500,L=45]
> (10) go polar_2d contour/pen=1/lev=(4800,6000,60)/overlay HGT[Z=500,L=45]
> (11) go polar_land " " 1.0
> (12) go polar_grid_fancy
> 
> 	   Line (7) must be repeated after (8); without doing that, the
> 	   land areas obscure the shaded areas. Is there any way that
> 	   the polar stereographic map can be plotted first and every-
> 	   thing else overlayed on that?
