On 6/22/2011 9:45 PM, michael hughes wrote:
Thanks Ansley, 

Making me think about the descriptor file helped.

After a bit a trial and error I found that my descriptor file was poorly defined. In the &BACKGROUND RECORD I had "D_TIME_UNIT = 432000", corresponding to 5 days in seconds, and each files' S_START and S_END numbered in sequence (0,1,2,3,4...). When I switched this to D_TIME_UNIT = 86400 (1 day in secs) and files numbered in units of 5 (0, 5, 10, 15...), it worked.

This was quite confusing given the descriptor file seemed to work (i.e. displayed data at correct times). It's only when I tried to transform the data that I ran into a problem.

On Thu, Jun 23, 2011 at 2:44 AM, Ansley Manke <Ansley.B.Manke@xxxxxxxx> wrote:
Hi -
What you show looks like the right set of commands. Have you checked the 5-day data from your descriptor dataset, making sure that the time range is correct and that there's data?  Making a few time-series plots or listing data at some locations is often a way to spot problems.

Do you see any messages when you go to use the regridded 1-day data?  If for instance your original data is on a different calendar-type, then Ferret won't do the regridding operation.

Ansley
