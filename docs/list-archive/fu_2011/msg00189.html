Hi all,
For information, I have found a bug with ncrcat because this command does not know the name "noleap". 
It accepts only the "no_leap" and produces no warning on this.
It is not conform to the CF convention so I wrote a bug report
http://sourceforge.net/tracker/?func=detail&aid=3217769&group_id=3331&atid=103331

To avoid problem, I wanted to use the name 365_DAY (also proposed by the CF)
but didn't find it in the documentation.
So, could ferret team add in the documentation
the 365_DAY  name to describe a NOLEAP calendar.
Note, that it seems to be already accepted by the define command.
yes? def axis/t="01-JAN-1900":"31-DEC-1920":1/units=years/edges/cal=365_DAY aa 
yes? show axis aa
name       axis              # pts   start                end
AA        TIME                21 r   02-JUL-1900 12:00    02-JUL-1920 12:00
T0 = 01-JAN-1900
CALENDAR = NOLEAP
  Axis span (to cell edges) = 21


To resume, authorized names for calendar are (with no case sensitivity):

|gregorian| or |standard|
|proleptic_gregorian|
|noleap| or |365_day|
|all_leap| or |366_day|
|360_day|
|julian|
