Hi all-
It turns out that Ferret was trying to create the message for a warning about the time axis information in the descriptor file. It does an internal write to a fixed-length string to include the warning message and the file name. This was fine with the shorter dataset name tyty, but ran into a write error trying to write the longer filename. 

This will be fixed in the next Ferret release.

Ansley

On 5/5/2011 11:17 AM, Ansley Manke wrote:

Hi Patrick,
The length of any dataset name is limited to 2048 characters, including names for descriptor files. I can't reproduce what you are seeing with Ferret v6.67. In fact I made a descriptor file with a name over 200 characters long and was able to open that successfully. 

Let's talk off-line and see if we can find out what is happening.

Ansley
