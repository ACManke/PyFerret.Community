Dear Ferret users, developers,

I'm trying to create a filled polygon of a land contour based on the
gshhs coastline dataset.  Apart from obvious problems around the 0
meridian, the idea works more or less.  I've got a coastline dataset
in netcdf comparable to the "geo_borders" dataset.

You can get the basic idea of what I'm trying to do with

> use geo_borders
> poly/fill /pal=gray continent_lon,continent_lat

This doesn't quite work because the polygons aren't closed, but that
can be resolved.  Greenland works OK.

The problem is that apparently my land contour data set is too large:
when I try this with the gshhs dataset, I either get 

> Too much polygon data to plot at once
> Too much polygon data to plot at once
> Too much polygon data to plot at once
> Too much polygon data to plot at once
> [repeated lots of times]

or I get a straight crash:

> yes? poly/fill coastline_lon,coastline_lat
> **ERROR Ferret crash; signal = 11

Using set memory to increase the available memory for ferret doesn't
seem to help.  It's probably memory related, as the error occurs
somewhat randomly and it did work on a different machine (same ferret
version) once.

Are there any settings I can play with to allow Ferret to draw larger
polygons?  Has anyone done this using gshhs or similar?

Kind regards,  
     Hein Zelle
