Hi Ravindra,

> While installing ferret in Ubuntu 10.10/Linux, i got following message while
> executing: source /usr/local/ferret_paths
> 
> No command 'setenv' found, did you mean:
>  Command 'netenv' from package 'netenv' (universe)
> setenv: command not found

sounds like you're trying to run the csh script while you're using the
bash shell.  The two shells are not compatible.  There is a file
ferret_paths_bash_template in the ferret/bin directory which does what
you want.  You just need to fill in the proper paths: the Finstall
script only fills in the csh version.
