Hi Tony,
The problem you face is that the days1900 function uses the default calendar, not the NOLEAP one.
The climatological_axes dataset contains a monthly NOLEAP climatological axis called MONTH_NOLEAP, so you can just use that. 

yes? use climatological_axes
*** NOTE: regarding /home/users/tmap/ferret/linux/go/climatological_axes.cdf ... *** NOTE: Climatological axes SEASONAL_REG, MONTH_REG, MONTH_IRREG, MONTH_GREGORIAN, MONTH_NOLEAP, MONTH_360_DAY defined
yes? list t[gt=MONTH_NOLEAP], tboxlo[gt=MONTH_NOLEAP], tboxhi[gt=MONTH_NOLEAP] 
             TIME: 01-JAN 00:00 to 01-JAN 00:00 NOLEAP
