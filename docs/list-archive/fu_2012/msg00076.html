> To get what you want, for data on a rectilinear grid, it would be
> 
>    shade/x=-180:180:30 temp[k=1]
> 
> and then the modulo character of the longitudes is applied. 

I have a similar problem with plotting coastlines, e.g. variables
coastline_lon and coastline_lat, plotted using plot /overlay /vs.  Just like
the "go land" script, I plot everything twice or even three times to
catch cases left of 0 or > 360.

Is there a way to make such data "wrap" in longitude so you only need
to plot them once?

It would be great if plot /vs either had an option (/modulo?), or would
be clever enough to detect it's plotting on a longitude/latitude grid
to do this automatically.
