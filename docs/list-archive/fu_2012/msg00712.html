Hi Martin,
I am not seeing this in an example which I created.  Is the output file being opened at all?  Can you tell whether the error occurs at some point in the script?  One experiment might be to write your script so that it writes to the output file after opening each one.  I realize that in a more complex script you might need the files open at the same time, and there is no reason Ferret shouldn't be able to do that.  The -script option is not putting Ferret into any special state, except for command-line reading.

Ansley
