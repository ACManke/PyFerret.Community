If you do this in the standard viewport notation, then you must overlap the definitions. In this case for example 

.../yli=.2,.5 t1
.../yli=.3,.6 t2
The problem here is that Ferret is assuming the need for large margins for its default labels.
But a better way is to use the DEF/VI/AXES syntax, that allows complete control of each viewport.
(And I can't think of any reason to have those extremely precise values for XLI/YLI. Who could possibly see the difference between 0.1998 and 0.2? Or between .5994 and .6?)
