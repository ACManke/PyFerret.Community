One way is to define some other (unused) pen number to be white:

ppl color,6,99,99,99   ! redefine pen 6 to be almost white

Now you can do thick white vectors:

vector/col=18 u,v
Note that I did not define the "white" color to be 100,100,100. I can't remember what the problem is, but I think it causes ps2pdf to crash. At least it did something bad many years ago, and since no one can tell 99 from 100, I've kept doing it.
By the way, the same thing works for white contours overlaid on a color-fill plot.
