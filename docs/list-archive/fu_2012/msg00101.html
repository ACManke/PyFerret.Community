On 3/17/2012 4:55 AM, Lev Tarasov wrote:
For anyone else exporting data to ARCgis with limited grid bounds crossing the prime meridion, ARCgis can't handle netcdf grids with longitudes outside of [0,360] range, contrary to netcdf standards (as 
Ansley mentions below). The crude solution I ended up with was rotating
the grid and having my GIS colleague (if only everybody used ferret...)
rotate it back within ARCgis..

On 3/5/2012 8:18 AM, Lev Tarasov wrote:

Hi all;
       I want to pass some ferret generated netcdf files to
a colleague who uses ARCgis, and the non-global grid range
brackets 0 longitude so that the netcdf grid ends up having
..359, 360, 361, 362... longitudes, which ARCgis can't handle.
Is there any easy way to make the longitude grid go ..359, 360, 1, 2..?
