hi,
When you specify uwnd[l=1:12:2@ave], you are defining a dynamic grid with a time axis that has every second point, and regridding to that grid. So you're probably doing more smoothing with that operation than you expected.  There's some discussion of the weights that are used for a regridding with @AVE, here: http://ferret.pmel.noaa.gov/Ferret/faq/regridding-a-time-series-with-ave

To take (point1+point2)/2, how about using @SHF:
yes? use monthly_navy_winds

yes? list/L=1:5 /x=180/y=0 uwnd,uwnd[L=@SHF:1], (uwnd+uwnd[L=@SHF:1])/2

             DATA SET: /home/users/tmap/ferret/linux/fer_dsets/data/monthly_navy_winds.cdf
             TIME: 01-JAN-1982 14:45 to 02-JUN-1982 19:15
             LONGITUDE: 180E
             LATITUDE: 0
 Column  1: UWND is ZONAL WIND (M/S)
 Column  2: UWND[T=@SHF:1] is ZONAL WIND (M/S)(shifted by 1 pts on T)
 Column  3: EX#3 is (UWND+UWND[L=@SHF:1])/2
                  UWND   UWND   EX#3
