Hi,
There seems to be something in the internal formatting of the file that means it isn't read correctly under Unix. Even  the unix "cat" command does not read the file.

   > cat tzclim.dat

returns nothing.  It sounds as if what Fisseha did was to do a "save as" text, from the email tool. I allowed the file to be opened by my text editor from the email attachment, and saved it as a text file from my editor. Then renamed it back to tzclim.dat.  That version of the file is attached.  Ferret does not pay attention to the file name for reading ascii files, so this has nothing to do with the naming.

   yes? FILE/VAR=height tzclim2.dat
   yes? plot height

(Sometimes a file that's written under Windows will have trouble being read under Linux.  That is not what happened here, but it seems to be something similar to that.)

Ansley
