Hi Sudherr,
This is because of the changes we made to adhere more strictly to the CF conventions, described in the Release Notes for Ferret v6.7.  See item #7 of:
http://ferret.pmel.noaa.gov/Ferret/documentation/release-notes/version-6-7-release-notes

Previously Ferret used the names of coordinate axes, such as the "depth" in these files, to determine the direction of axes, and whether a vertical axis is oriented positive-up or positive-down.  Now we use the attributes in the CF conventions, and also the CF standard name attribute (linked here, http://cf-pcmdi.llnl.gov/).  which requires that a depth axis would have an attribute on the depth axis of

                depth:standard_name= "depth" ;
or
                depth:positive= "down" ;

--Ansley
