Hi Murat,
This will be improved with the next release of Ferret. Currently, Ferret translates all commands and variable names to uppercase, which allows the user to specify commands in either upper or lower case.
In the next release, all of the information about variables including their names exactly as they appear in the file, and all attributes, will be saved when the file is read. On output, the variable names in the original upper or lower case will be written, and the user can specify whether to write out attributes which were in the file but are not the standard attributes that Ferret always writes.

Until that release is ready, our best suggestion is to use NC operatoras to fix the file after it has been written out. These are available at http://nco.sourceforge.net/

Ansley
