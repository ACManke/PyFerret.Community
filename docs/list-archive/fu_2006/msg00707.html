Hi Jimmy,

It is possible to create netcdf files > 2Gb using the classic netcdf interface. See http://www.unidata.ucar.edu/software/netcdf/docs/netcdf/Large-File-Support.html for more details. I guess you're doing it this way?

However, this still uses the 32bit offset but I specifically want to change the netcdf mode to 64bit which you do by passing a flag to NC_CREATE. This changes the magic number at the start of the file which may break netcdf savvy software if it's linked against older versions of the netcdf library.

This assumes of course your OS support large files.

Glenn

On Fri, 3 Nov 2006, Ansley Manke wrote:

Hi Glenn,
Ferret gets its netCDF libraries through the OPeNDAP libraries, which at this time don't support all of the capabilities of the Unidata netCDF Version 3.6.0 (which is able to support the file format with 64-bit offsets for very large files).

The OPeNDAP folks are working on this now, and I hope it won't be too much longer before we'll be able to offer this in Ferret as well.

Ansley
