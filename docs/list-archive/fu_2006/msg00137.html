Emilie,

I think the best path from a netCDF file into Vis5D is actually through the Integrated Data Viewer (IDV) (http://www.unidata.ucar.edu/software/idv/) from Unidata. The creators of Vis5D have been concentrating much of their development effort on an all-Java product called visAD. The IDV builds on top of the visAD effort by adding on a complete user interface, many standard 3D plot types and the ability to read many different scientific data formats including netCDF.

If it were me, I'd skip writing the vis5D file (and using Vis5D) and go straight to using IDV with the netCDF file.

Roland
