Hi Glenn,
Ferret gets its netCDF libraries through the OPeNDAP libraries, which at this time don't support all of the capabilities of the Unidata netCDF Version 3.6.0 (which is able to support the file format with 64-bit offsets for very large files).

The OPeNDAP folks are working on this now, and I hope it won't be too much longer before we'll be able to offer this in Ferret as well.

Ansley


Glenn Carver wrote:
I'm doing some high resolution modelling which is resulting in problems with the file sizes and I may need to generate netcdf files > 2Gb by enabling '64bit offset' mode when creating netcdf output files from the model.

I was wondering whether the 64bit offset netcdf files are handled by Ferret? (obviously I haven't tried it 8-). I had a read of the manual and tried a google but couldn't find the answer? I note there's a 64bit linux ferret binary but does it understand 64bit netcdf files? Are there other 64bit binaries?
