Hi Ferret users,

| I am using Xandros 2.0 Linux. I installed Ferret using
| a shell script written by someone else. When I execute
| the "ferret" command from the prompt, I get an error.

In addition to the two options Jaison suggested,
there's another possible solution, which I think involves
more efforts than Jaison's first solution (Install the
statically linked version) but possibly less than his
second (Build ferret from scratch).

| "ferret: error while loading shared libraries:
| libtermcap.so.2: cannot open shared object file: No
| such file or directory"

I had a similar problem in the past with Debian Linux.
Basically, what you would do is
1) to find libtermcap.so.2 on your system,
2) to use a compatible library on your system, or
3) to install libtermcap.so.2 .

Since I didn't found a compatible library on my system,
I did (3): I searched the Internet, found an RPM
of the missing library, and installed it.  If you need
further assistance, please write to me.
