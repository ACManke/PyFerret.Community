Steve,

I run ferret up to 5.70 on RH7.3 without problems. (No luck with either
precompiled version of ferret5.80 - it looks to me like the binaries
were compiled with libstdc++ versions that are newer than those for
RH7.3.)

What's your glibc version? I have ("rpm -q glibc"):
glibc-2.2.5-4

If your version is older, you can get the RPM at www.rpmfind.net

I don't use the shared object library tar file that was supplied with
earlier versions of ferret (5.4x).

You can use the command
$   ldd ferret 
in your directory /usr/local/ferret/bin to see the required libraries.
The libc one should be
	libc.so.6 => /lib/i686/libc.so.6 (0x42000000)

and
$   ls -l /lib/i686/libc.so.6 
lrwxrwxrwx    1 root     root           13 Dec  1  2003
/lib/i686/libc.so.6 -> libc-2.2.5.so
