Hi Carol,
This is indeed a bug, in some code intended to fix another bug having to do with the precision of the labels on color keys.It comes up only with the SHAKEY setting klab_dig=0 (argument 5)

This'll be fixed in the next release, but a workaround is to let the 5th argument to SHAKEY take its default value, or set it to something other than 0.

   fill/k=1/l=1/lev=(-inf)(-5,5,1)(inf)/key/set temp
   ppl shakey 1,0,.16,0,  ,4,1,7,5.1,5.4
   ppl fill
