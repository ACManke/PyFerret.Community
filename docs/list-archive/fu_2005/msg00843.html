Hi Yogesh and Ansley,

Dan Holloway looked at this data set a bit more and he thinks that I may have been wrong when I said earlier that there seems to be a server problem associated with it. It does fail in the ODC, but he thinks that that is because of the number of variables in the data set, i.e., it is a problem in the ODC, not with the server. The error below is likely because you forgot the .ascii.

Ansley, is there any way that you can check to see if Ferret can handle the dds that the server is returning. It is VERY big and quite convoluted, although it appears to be legitimate. Try

http://g0dug03u.ecs.nasa.gov/dods-bin/nph-dods/datapool/MOAA/ MYD08_M3.004/2003.05.01/MYD08_M3.A2003121.004.2004136064318.hdf.dds
