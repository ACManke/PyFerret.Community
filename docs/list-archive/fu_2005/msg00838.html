On Fri, 2 Sep 2005, Kevin O'Brien wrote:
Hi Steve -

If you are getting that error message, chances are that you're not invoking the static version of Ferret. Try doing a "which ferret" and see what executable is listed . My guess is that by default you are invoking the shared-library version of Ferret. If you want the default "ferret"command to be the static version, you'll want to make sure $FER_DIR/bin/ferret is a link to the static version (which for v581 would by default be called ferret_v581_static).



Kevin
