  Regarding your problems:

 1. The message you mensioned is just an informatiove one..nothing serious.
       For vertical axes Ferret expects units like meters, centimeters, km,
       feet, level (for a full list please see Ferret documentation : the last
       part of Ref Sec4.2.  DEFINE AXIS). If Ferret encounters an unknown unit
       like "eta_level", it issues this warning to tell that when computing axis
       transformations, abstract index will be used. If you want to get rid
       of this message you have to regrid the data to a Z-axis with acceptable
       units.

 2. Your data seems to be on curvilienear grid. To convert the axes to lat and
       lon, you have to use the interested variable with lat(j, i) & long(j, i)
       from the NetCDF file in Ferret external functions like CURV_TO_RECT.
       You can find detailed example and explanations in the Ferret documenta-
       tion: Appendix A Sec11.  CURV_TO_RECT.
