It turned out that John was running an older version of Ferret which didn't have all the string capabilities that the more recent ones do. Saving string variables to netcdf files works fine.

Ansley

John Luick wrote:

If I have a string variable jprj, such that:
yes? list jprj
VARIABLE : Map projection type
FILENAME : .\tvl_grd.nc
"ME"
...the command:
save/clobber/heading=enhanced/file=test.nc jprj ...causes ferret to crash. If jprj were a regular sort of
string, like one created by a="string", this command could
save it. But it doesn't work with jprj, which is a normal
garden-variety sort of character string found in netcdf files
the world over. It was read in by ferret when it read tvl_grd.nc.

The netcdf file has about 15 variables in it of various sorts
(mostly numerics, a few strings).
