James Orr wrote:

> I use Ferret 5.8, and this script works just fine (as it did for
> .plt files made with previous versions of Ferret).  Sounds like the 
> problem lies in other software on your system.

You are correct: the problem is in the version of 'gs' being used by
ps2epsi. At home I had two versions installed: gs 7.07 (gs-esp as it
is called by debian) and gs 8.01 (gs-gpl). The default, 7.07, works
correctly. At work I had only installed 8.01 which does NOT work
correctly. I solved it by also installing gs 7.07 and can now generate
properly cropped EPS files again.

I will try out the other suggestions in this thread about modifying
the postscript - if those work properly a modification to Fprint to
produce EPS files by default (or optionally) may be the best solution.
