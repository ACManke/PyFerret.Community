Hello,

I have Ferret 5.6 running under Linux Fedora Core 1. Running a batch job
like:

  ferret -script try.jnl

works fine.

However, on another machine I run Ferret 5.7 running under Linux Fedora 
Core 2. In this case:

  ferret -script try.jnl

doesn't work (it just opens a Ferret session without running try.jnl). 
Even:

  ferret -h

doesn't work -- i.e. it doesn't give the "help" information.

However, on the second machine:

  ferret < try.jnl

works fine (except that I need to have a "quit" at the end of the file 
try.jnl, or otherwise ferret just hangs).

What is the reason for this different behaviour? May I just have a bad 
installation of Ferret 5.7 on the second machine?
