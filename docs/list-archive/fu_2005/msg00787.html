Herve,

It should not be necessary to convert your profile data to NetCDF. If your goal is to use L.A.S. (and I'll assume this refers to LAS, the Live Access Server), then I suggest starting with the LAS documentation regarding in-situ or non-gridded data stored in a relational database.
http://ferret.wrc.noaa.gov/Ferret/LAS/FAQ/relational_database_access.htm

This functionality was originally developed to work with the JGOFS DBMS, but works with MySQL DBMS or there are instructions for writing your own interface module for reading in data from any relational database.

good luck!

When you run into the next set of questions, you might try posting your question to LAS_users. Ferret does a lot of the behind-the-scenes work for LAS, but for help getting started using the in-situ capabilities of LAS, you would want to post questions to the LAS_users list instead of ferret_users.

To check out that listserv, see
http://ferret.pmel.noaa.gov/Ferret/LAS/Mail_Archives/

Cyndy
