Hi Lev,
It looks like you have uncovered a bug in Ferret's ASCII reading  with the F axis. I think it's more with the /ORDER qualifier than /COL. I see the same bug using Ferret v6.93. Testing with your short portion of the dataset, and defining a grid just in XF,
yes? DEFINE AXIS/F=1:12:1 T12
yes? DEFINE AXIS/X=1:4:1 x4ax
yes? DEFINE GRID/X=x4ax/F=T12 xfgrid
yes? FILE/SKIP=4/VAR="Tm"/COL=12/order=fx/GRID=xfgrid fxdata.dat
yes? sh dat
     currently SET data sets:
    1> ./fxdata.dat  (default)
 name     title                             I         J         K         L         M         N
 TM       TM                               1:4       ...       ...       ...       ...       1:12
 
yes? list tm
             VARIABLE : TM
             FILENAME : fxdata.dat
             SUBSET   : 4 points (X)
             F        : 1
 1   / 1: -8.090
 2   / 2: -8.090
 3   / 3: -8.090
 4   / 4: -8.090
Though it reads all of the data if I leave off the /ORDER=FX, and the commands you used work correctly on an x-T rather than an x-F grid.

We'll check into this.
