Hi Takaya,
I think this must have to do with the way the contour command works.
The script computes a new mask for the data and I think there must
be something in the script that is not computing the new mask in enough
detail.   I will look at it when I have more time, but I wonder if using shade
would give you what you want?

yes? USE etopo5
yes? SET region/x=270:285/y=25:30
yes? LET ocean_mask = IF rose LT 0 then 1 else 0
yes? SHADE/PALETTE=ocean_blue  rose
yes? SHADE/OVER/PALETTE=gray ocean_mask

This overlays just as in the script fland, so you could do the same thing by
making your plot and then saying:

yes? go fland 5 gray

> > > > > >
> > > > > >The qualifiers /LEVELS=(0,10000,10000)/PALETTE=gray tell Ferret to
> > > fill in
> > > > > >gray everywhere that the elevation exceeds zero (i.e. land).
> > > > > >
> > > > > >For your application you need only replace the "0" with the
> desired
> > > depth.
> > > > > >For example,
> > > > > >
> > > > > >      set data etopo60
> > > > > >      shade/lev="(-500)(10000)"/pal=gray rose
> > > > > >      go land
> > > > > >
> > > > > >will create a mask of all areas shallower than 500 meters ("go
> land"
> > > > > overlays
