Hi all,

Just to finish this thread, it turns out that the file has a timestep that can't be turned into a valid date.  Using "ncdump -c" to look at the coordinates of the data, shows the time coordinate as

It shows that time is expressed as days since 2019-01-06

        float time(time) ;
                time:long_name = "time" ;
                time:units = "days since 2019-01-06 12:00" ;

but the time coordinate value is listed as the value:

 time = -2.91164e+25 ;

and that value cannot be transformed into the internal representation of time within PyFerret/ Ferret.

-Ansley
