Dear pyferreters,

I do have an installation of pyferret 7.6.3 on a Linux-Cluster, which runs smoothly under python 3.7.7.

Now, I installed the same pyferret version on a second (somewhat newer) Linux-Cluster under python 3.9.7.
Starting pyferret, I immediately receive the error message:

ImportError: cannot import name 'libpyferret' from partially initialized module 'pyferret' (most likely due to a circular import) (/path/to/PyFerret-7.6.3-RHEL7-Python3.6/lib/python3.6/site-packages/pyferret/__init__.py)

Is that a known issue and is there a fix or workaround for this?
Any help is very much appreciated.
