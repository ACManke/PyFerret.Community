Hi,
I see the behavior you're reporting, but only with Ferret v7.3.  Defining a similar grid and a variable on the grid:

yes? define axis/t="15-AUG-2016 00:00":"15-AUG-2016 11:00":3600/units=seconds/t0=1-jan-1900 time
yes? define axis/z=1:58:1/depth/units=meters zaxis
yes? let var = L[gt=time] *RANDU(z[gz=zaxis])

yes? shade var

Is indeed giving a blank plot with v7.3.  The plot is working fine with more-recent versions of Ferret, so there must have been an undiscovered bug in that version.  If you update you should be fine.  We are working on a release of Ferret v7.5 by the end of this week, so you might wait a few days for that.

By the way, you could redefine the time axis to have units of hours, and it behaves fine:

yes? use myfile.nc
yes? define axis/t="15-AUG-2016 00:00":"15-AUG-2016 11:00":1/units=hours/t0=1-jan-1900 `var,return=taxis`
Replacing definition of axis TIME
           *** NOTE: grid GSI1 used by data set mfile
           *** NOTE: Redefinition may alter apparent contents of data set
yes? shade var
