Hi Ferreters,

I wonder if it is (will be) possible to make Ferret NOT honor certain attributes of a remote opendap dataset, esp. when the attributes are not well compatible with Ferret. For example, Ferret cannot read the vertical axis of this following dataset correctly:

yes? use "http://iridl.ldeo.columbia.edu/SOURCES/.NOAA/.NODC/.WOA05/.Grid-1x1/.Annual/.an/dods"
yes? show grid temperature
    GRID GCR2
 name       axis              # pts   start                end
 X         LONGITUDE          360mr   0.5E                 0.5W
 Y         LATITUDE           180 r   89.5S                89.5N
 Z1        Z                   33 r   1                    33
 normal    T

This is because, I suppose, the erroneous bound attribute of the vertical axis that points to itself:

Z:bounds = "Z" ;

This can probably be corrected ad hoc. But would it be nice to have Ferret ignore this attribute?

Thanks,
David
