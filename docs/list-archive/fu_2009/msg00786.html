Hi Nuncio,
The problem is similar to the one you found in the archives, but in your case the incorrect value among the coordinates is too large rather than too small. If you look at the coordinate values in the longitude variable, they start with: 

 NbLongitudes = 589.999999999977, 230.33333333331, 230.666666666644, ...
So the first longitude is the one that's out of order. The fix would be similar to the one outlined in the Archive message, but the definition of the new coordinates axes would be: if the value is greater than 360, subtract 360. 

  LET fix_x = IF ($xaxname) GT 360 THEN ($xaxname) - 360 ELSE ($xaxname)
