Hi Ansley,

Thank you for the detailed explanation.

| [. . .]
| Essentially all the capabilities needed are now included in the
| BOUNDS syntax, but EDGES are in Ferret for backwards compatibility.
| 
| [. . .]
| Because BOUNDS are in the CF convention, they are
| probably more likely to be recognized by other software
| than EDGES would be.

So, I guess the bottom line is

1) You don't have to edit existing netCDF files if
   you work only with Ferret;

2) But, if you create new netCDF files, use BOUNDS,
   because it's potentially more compatible with other
   packages.
