Hi Paulo,
This error is coming to Ferret from the OPeNDAP libraries, which we use to read netcdf files, and the data in descriptor files which are in sets of netcdf files.
The error code is translated by the OPeNDAP library as "Input/output error", which isn't too helpful here. Are you seeing this error when you go to use data, that is on a PLOT or LIST kind of command; or is it when you open the dataset? Are you trying to SAVE data? Is it possible be that the stepfiles (the netcdf files that are listed inside the descriptor file) are sometimes changed, or are not available? I'm guessing here - a stepfile that is not available should result in a different error message. 

Ansley
