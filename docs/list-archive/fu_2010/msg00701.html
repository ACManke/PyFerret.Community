Hi all,
Sudheer investigated this further and it turns out that with the 64-bit executable ferret_v664, the renaming of ferret.jnl to ferret.jnl.~*  fails when the directory contains a large number of previous versions of ferret.jnl. This seems to be an issue with some of the shared system libraries that ferret_v664 links with. The next release of Ferret will be linked with static libraries wherever possible, and the behavior is fixed.

Ansley
