Hi Andrew,
I was going to suggest the same thing that apparently Wolfgang did.  Just to be sure, I'll go through some of the details which may come in  handy for others.

Look at the value of the environment variable PLOTFONTS.  It should point to the directory ppl/fonts that came from the new distribution. In the directory where you unpacked the fer_environment.tar.gz file, the directory will look like this:
/home/porter/ansley/ferret> tar xzf fer_environment.tar.gz
/home/porter/ansley/ferret> ls -laF
total 7616
drwxr-xr-x  5 ansley tmap     4096 Jan  7 10:16 bin
drwxr-xr-x  2 ansley tmap     4096 Dec 10 17:25 contrib
drwxr-xr-x  2 ansley tmap     4096 Mar 13  2009 doc
drwxr-xr-x  2 ansley tmap     4096 Dec 10 17:25 examples
drwxr-xr-x  3 ansley tmap     4096 Dec 10 17:05 ext_func
-rwxr-xr-x  1 ansley users 7741987 Jan  8 08:49 fer_environment.tar.gz
drwxr-xr-x  2 ansley tmap     8192 Dec 10 17:25 go
drwxr-xr-x  2 ansley tmap     4096 Mar 13  2009 gui
drwxr-xr-x  3 ansley tmap     8192 Dec 10 17:08 ppl
The environment variable PLOTFONTS should point to
/home/porter/ansley/ferret/ppl/fonts/
Referring to the Installation Guide, The Finstall script needs to be run with option 1 to "install executables" and then with option 2 to "Customize ferret_paths". That second run through will write the correct location into the ferret_paths script. Execute ferret_paths

> source ferret_paths

and then a call to run ferret_paths needs to be in .login script (or the appropriate method for doing that if you're running a shell other than csh).

Ansley
