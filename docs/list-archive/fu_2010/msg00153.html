Hi Peter,
Yes, this looks like a bit of a bug.  It also occurs in earlier versions of Ferret, so has been in the code all along.  It looks like it comes from the option to
SET LIST/FILE=AUTO  or
LIST/FILE=AUTO
which generates a filename from the input dataset and variable name. When Ferret parses command lines, options are generally just compared up to the first four characters; but here it might be a good idea to see if there's more after the letters "AUTO".  You would not run into this if you used, say,
list/noheader/file=tautocor_nino_INDO.dat/
Thank you for the report and the example.

Ansley
