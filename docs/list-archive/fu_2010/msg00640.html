Dear ferreters,
in our default model output, the y axis (latitude) is unconventionally defined from north to south. Working with the unchanged file in ferret seems to work fine. But since ferret does not recognize this unconventional axis/grid correctly, it produces a mirror image of the data with respect to latitude when for example appending a variable to the file. This "mirroring" happens without error notice or warning, which can of course lead to misunderstandings interpreting the data. 

I prepared an example, see attachment.

Thank you in advance for taking notice.

Best wishes,
