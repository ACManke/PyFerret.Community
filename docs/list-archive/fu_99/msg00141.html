Yes, I have encountered the same difficulty with labels for vector
overlays.

I've also not found a way to resize the vector label (the vector key).
One way round this is to resuse fland.jnl to redraw the map-fill over the
vector key, then draw a vector and write an explicit label; but this
does not work if the label has to be drawn outside the plotting area, as
is often necessary when using an idealised, rectangular domain with no
map details.

The veckey command does not provide as much user-level control as the
shakey command for contours.  I don't know if there is a workaround.
