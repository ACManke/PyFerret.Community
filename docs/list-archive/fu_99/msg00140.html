Ferreters,

one of the things I have found frustrating about ferret is its
interaction of labeling with the options set_up and overlay in
several plotting commands. As an example, consider that one wants
to overlay a velocity vector field over a pressure contour map.
This can be nicely accomplished by contouring the pressure field
and then overlaying the vector field with something like:

yes? contour pressure
yes? vector/asp/over u,v

Suppose now that you want to get rid of some of ferret's default
labeling. You would simply change the above to:

yes? contour/nolab pressure
yes? vector/asp/over/nolab u,v

This omits all labeling except the desired axes labels and the
line describing the aspect ratio adjustment for the vector field.
Suppose you don't want this adjustment label on the plot. The
only way I know of removing undesired default text on a plot is
to do it manually with the set_up, go unlabel n, ppl vector
sequence. But this doesn't do the trick in the above scenario.
The problem lies in the final ppl vector which replots the
vector field and clears the base pressure map.

Anyone run across this one before?

David
