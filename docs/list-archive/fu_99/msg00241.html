Thanks to David Darr, Gabriel Vecchi and Halldor Bjornsson for answering the
question I posted yesterday about how to read MOM output into ferret. I'll
pass along David's response so it's in the searchable email archive.

It's probably only news to ME that the way to read a netCDF file into Ferret
is by typing "use filename." I've always preferred Ferret -- now I'm sold on
netCDF!

David Darr wrote:

> > Fellow Ferreters: I've been using Ferret for years, but
> > reading only ascii data files. Now I want to use it to:
> >
> > 1) plot netcdf output from the GFDL MOM model
> > and
> > 2) write data as input to the model
> >
> > Is there anyone out there who does this and can email
> > me a ferret script or two to start with? One that reads
> > the snapshots.dta.nc diagnostic, for example?
>
> Karen -- Forgive me if I am misinterpreting your questions -- but 1) is
> quite simple. Ferret deals with netCDF in a much more transparent fashion
> than it does with ascii. To read snapshots.dta.nc for example, just do:
>
> yes? use snapshots.dta.nc
>
> and then
>
> yes? show data
>
> to see what you have. THen you proceed as you would with any other data
> set in Ferret.
