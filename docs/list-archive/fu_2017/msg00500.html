Yes, if the axes in the NetCDF file can be identified as longitude, latitude, level (height or depth), or time, Ferret will automatically reorder so the X/I is longitude, Y/J is latitude, Z/k is level, and T/l is time.  See "Files with permuted axis ordering" in 
  http://ferret.pmel.noaa.gov/Ferret/documentation/users-guide/data-set-basics/NETCDF-DATA
