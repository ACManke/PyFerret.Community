Hi, just a guess, in which direction one may search for the error source.

- A20160012016366: this seems to be a file which is not very old

- http://www.unidata.ucar.edu/software/netcdf/docs/nc-error-codes.html   --> you find the error message here. NC_EHDFERR - is not very specific for me, but the error seems to be related to hdf-5 encoding of the netcdf4 format. The webpage is from 2016. I would suggest to check, if your ferret and especially the netcdf-library linked in is able to deal with this version of the netcdf format. I am not sure how to do this, but the first step would be to check the version of ferret.

Alternatively you may investigate the properties of the file with a ncdump, which comes with the netcdf-library. See the man - page of ncdump for details. Here you know the version of the netcdf-library.

Best,

Martin
