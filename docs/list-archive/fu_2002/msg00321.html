Hello,

Glenn Carver reported a problem loading netcdf files on 20 august
1999. If a netcdf file 'testcase.nc' is in the FER_DATA searchpath,
and it is opened using

use testcase

ferret reports

 **TMAP ERR: non-existent or not on line
             Searched $FER_DESCR and $FER_DATA for testcase(.des,.cdf,.nc)

If the name of the file is changed to 'testcase.cdf', everything works
as expected. While Glenn found this problem in Ferret 5.00 on Solaris
2.5, I now see it in Ferret 5.22 on IRIX 6.5, and also in 5.40 on
SunOS 5.8. Is there a solution for this problem? Should I report it as
a (small) bug?
