Hello,

I'm not sure if someone else replied already. What I suspect is
happening is that there's vector data in the HDF files that are
unrecognizable. It's actually not the server that's not recognizing
them, it's the combination of ferret and the DODS-netCDF client library
(which ferret is using to read data from DODS servers). DODS servers
encode vector data in a datatype called a Sequence but the netCDF client
library does not (yet) know how to read and process that datatype. The
next release of our software will address this problem.
