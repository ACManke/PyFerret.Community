Hi Xin,
As Andrew says, this is fixed in version 5.41.  I just want to add that
In both 5.4 and 5.41  you can control the axes  with the /AXES=
qualifier, restoring all or some of the axes:

yes? set view ul; shade/noaxes sst
yes? set view ur; shade/axes=1,1,1,1  sst

Ansley Manke

Xin Jin wrote:

Hi Ferreters,

In ferret version 5.4  after using noaxes option in shade command, the axes are missing when you use the command again.  For example

shade/noaxes  sst : there are no axes. This is what you want. Then

shade sst : There are no axes too.  However you really want the axes.
