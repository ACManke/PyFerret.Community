The latest version of DODS (3.2) supports authenticating proxy servers by
embedding a username and password into the DODS URL:

http://<proxy-username>:<proxy-password>@<dods-host>/<dods-dataset-path>

This doesn't seem to work with ferret beta 5.33, which doesn't surprise me
because the DODS 3.2 libraries were only released on 10 Dec, 2001.

However, I'd just like to confirm whether the next ferret release will
link against the new DODS libraries and therefore support this proxy
server authentication? Any ideas when the next ferret release might be?
