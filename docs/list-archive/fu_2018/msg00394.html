Did you upgraded to Ubuntu 18.04? It happened to me as well. For some reason your readline library got messed up. You probably have libhistory.so.7. You can fix it using a symlink

 

cd /lib/x86_64-linux-gnu

sudo ln -s libhistory.so.7.0 libhistory.so.6


but it might break ferret as it happens to me. Currently, I am trying to install from the beginning ferret.

Hi Subhrajit,

Can you also tell us what operating system (OS) and what version of ferret you are using? libhistory.so.6 looks like a dependency file to me (I am not an expert at all but met similar problems before). Go to /usr directory (cd /usr in terminal),  search inside /lib, /local/lib whether libhistory.so.6 is there. If so, link it to the path that ferret was looking for. If not, download and install one (you can search this file with your computer OS online and install it). Be prepared that ferret may ask for more dependencies. 😹
