I am trying to access a data set via opendap. The server requires username and password. I have them both.

However, the username is my e-mail address and containes an "@".

So the syntax

use http://username:password@xxxxxxxxxxxxxxxxxx/path/to/file.nc

does not work. I tried to bind like "email":"password" but with little success. Is there a way to use an e-mail address as a user name?

I'm not sure at all if this helps, but the same question has been asked in a more general context:

    https://stackoverflow.com/questions/10050877/url-username-with

The answer in the above link is, encode the @ symbol if you want to prevent it from being interpreted.
